# 导出图片段落标记显示修复设计

## 问题

谱面导出时，段落标记字母在 html2canvas 中发生基线偏移，导致约一半文字落到方框之外；同一标记在页面预览中显示正常。将导出外层从按钮改为非交互元素后问题仍存在，说明偏移来自字母文本的导出基线计算，而不是外层元素语义。

## 方案

- 保留现有导出 `div` 与普通页面 `button` 的外层分支。
- 在段落方框内为字母增加独立内层元素。
- 仅在导出状态下将字母相对上移 `0.5em`，即自身字高的一半；方框位置和尺寸保持不变。
- 顶部乐谱信息区“演奏顺序”的段落方框采用相同的字母内层和导出位移；箭头及方框保持原位。
- 页面预览不应用位移，继续保持现有视觉位置和点击定位能力。
- 不在通用导出工具中扫描或替换谱面 DOM。

## 验证

- 添加源代码级回归检查，验证谱面内部与顶部“演奏顺序”的字母均仅在导出状态应用 `-0.5em` 位移。
- 运行 ESLint 和生产构建。
- 不启动本地服务器，由项目维护者执行最终导出图片视觉验证。
