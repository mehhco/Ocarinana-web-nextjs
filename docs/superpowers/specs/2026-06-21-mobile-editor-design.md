# 移动端编辑器优化设计

## 1. 背景与目标

当前编辑器以桌面端双栏工作区为基础，工具栏和主体均存在 `min-width: 1000px`，移动端窄屏会发生内容裁切。现有按钮、歌词输入框和帮助提示也主要围绕鼠标与键盘设计。

本次优化以竖屏单手编辑为主要场景，同时增强横屏体验，并保持桌面端现有操作习惯。目标是让用户可以在手机上完成以下核心闭环：

1. 新建或打开乐谱。
2. 查看并选择谱面元素。
3. 连续录入音符和休止符。
4. 修改时值、附点、延长线和歌词。
5. 撤销、恢复、保存和导出。

首期不包含双指缩放、拖拽排序、播放功能或独立移动端路由。

## 2. 设计原则

- **谱面优先**：移动端主区域始终优先展示谱面，不长期占用空间展示完整工具面板。
- **高频常驻**：数字音符、休止、删除和撤销等高频操作在一至两次点击内完成。
- **渐进展开**：低频功能放入底部抽屉，避免主界面信息过载。
- **状态共享**：桌面端和移动端复用 Zustand store、谱面组件和业务动作，不维护两套编辑逻辑。
- **触控可靠**：交互目标最小尺寸为 44×44px，避免依赖 hover，明确处理滚动和点击冲突。
- **改造可控**：以响应式组件组合替代独立页面，优先解决可用性，再处理高级手势。

## 3. 响应式工作区

### 3.1 断点策略

- 小于 `768px`：移动端工作区。
- `768px` 至 `1023px`：平板/横屏工作区，使用移动工作区的增强布局。
- `1024px` 及以上：保留现有桌面双栏工作区。

CSS 负责布局切换，React 不根据首屏 `window.innerWidth` 条件渲染，避免水合差异。只在确实需要行为差异时使用 `matchMedia`。

### 3.2 移动端页面结构

移动端按垂直方向划分为三个区域：

1. **顶部栏**：返回、标题入口、保存状态、撤销/恢复和更多操作。
2. **谱面区**：占据剩余可用高度，负责浏览、选择和歌词输入。
3. **底部输入区**：常驻快捷输入条；需要更多功能时展开底部抽屉。

工作区高度使用动态视口单位，并包含安全区：

- 页面主体使用 `100dvh`，保留 `100vh` 作为兼容回退。
- 顶部和底部区域分别叠加 `env(safe-area-inset-top)` 与 `env(safe-area-inset-bottom)`。
- 软键盘弹出时，谱面区允许收缩，底部抽屉不与歌词输入框互相遮挡。

### 3.3 横屏与平板

横屏和中等宽度设备继续使用移动交互模型，但可以扩大底部输入区：

- 数字音符和常用修饰操作可并排展示。
- 底部抽屉最大宽度居中，避免内容横向拉伸。
- 不恢复桌面双栏，直到达到桌面断点。

## 4. 顶部栏

移动顶部栏保持单行，按钮触控区域不小于 44px：

- 左侧：返回按钮。
- 中间：可截断的乐谱标题；点击打开“乐谱设置”抽屉。
- 右侧：撤销、恢复、保存状态/保存按钮和更多菜单。

“更多”菜单包含导出图片、显示/隐藏指法图、显示/隐藏歌词等全局操作。保存状态采用图标加短文本呈现，例如“保存中”“已保存”“未保存”，失败时继续使用现有 Toast 错误提示。

桌面端现有完整工具栏保持不变，但应移除对移动端造成裁切的共享最小宽度限制。

## 5. 底部输入系统

### 5.1 常驻输入条

默认显示以下操作：

- 音符 `1–7`，根据当前陶笛音域禁用不可用音符。
- 高音/中音/低音切换。切换后数字键保持原位置，减少手指移动。
- 休止符快捷入口。
- 删除当前选中元素。
- “工具”按钮，用于展开完整底部抽屉。

音符按钮点击后沿用现有替换/插入规则。按钮提供按下态和禁用态，不再依赖 hover 表达反馈。

### 5.2 上下文操作条

当用户选中谱面元素时，常驻输入条上方显示一行上下文操作：

- 音符：时值、附点、延长线、歌词、删除。
- 休止符：时值、删除。
- 延长线或小节线：适用于该元素的操作和删除。

上下文操作仅显示当前元素可执行的动作，减少用户进入完整工具抽屉的次数。

### 5.3 工具抽屉

底部抽屉按功能分组：

- 音高与输入方式：高/中/低音、替换/插入。
- 时值与节奏：时值、附点、延长线、休止符、合并时值线。
- 结构：小节线、反复记号、连音线、段落结构。
- 表达：力度、换气、装饰音。
- 歌词：显示歌词、第二行歌词、清空歌词。

抽屉初始高度不超过可视区域的 60%，内部独立纵向滚动，并支持展开到接近全屏。首期使用明确的关闭按钮和遮罩点击关闭，不实现依赖第三方手势库的拖拽吸附。

危险操作“清空歌词”需要二次确认，避免移动端误触。

## 6. 乐谱设置抽屉

点击顶部标题打开设置抽屉，包含：

- 乐谱标题。
- 调号、拍号、速度及速度显示开关。
- 制谱、作词、作曲和其他信息。
- 当前陶笛类型的只读展示。

表单输入字号至少 16px。抽屉关闭时自动保留 store 中的变更，云端保存继续沿用现有自动保存与手动保存逻辑。

## 7. 谱面交互

### 7.1 选择与滚动

- 谱面区默认只允许纵向滚动，避免整个页面产生多层滚动。
- 音符的可视宽度可以保持原有比例，但实际触控区域最小为 44px。
- 小节线和段落标记通过透明触控外框扩大点击区域，不改变导出结果。
- 选中状态使用背景、描边和轻微触觉式按下反馈，不只依赖颜色。
- 点击谱面空白区域清除选择；处于连音线或合并时值线模式时，空白点击取消当前模式。

首期不添加长按菜单，避免与浏览器文本选择和系统菜单冲突。

### 7.2 歌词输入

- 输入框字号不低于 16px，触控高度不低于 44px，但导出态仍使用现有谱面尺寸。
- 设置 `enterKeyHint="next"`，确认后移动到下一个可输入音符。
- 保留中文输入法 composition 处理和批量粘贴逻辑。
- 输入框聚焦后确保自动滚动到软键盘上方；滚动只发生在谱面容器内。

### 7.3 帮助内容

移动端不使用 Tooltip 作为主要帮助入口。工具抽屉中的每个分组提供简短说明；需要完整解释时点击信息按钮，在抽屉内展开帮助。桌面端可继续使用现有 Tooltip。

## 8. 性能优化

性能工作与响应式改造同时进行，但不引入谱面虚拟化：

1. 将 `ScoreCanvas` 对整个 Zustand store 的订阅拆分为精确 selector，避免保存状态等无关变化触发整张谱面重渲染。
2. 保持小节与音符组件的稳定 props，避免在渲染过程中反复创建会破坏 memo 的回调和集合。
3. 指法图继续使用现有静态 SVG，但只为当前需要展示的音符创建图片节点；隐藏指法图时不挂载图片。
4. 将 `html2canvas` 从根布局移除，在用户点击导出时动态加载，并提供加载失败提示。
5. 将 Google Font 的 CSS `@import` 替换为 Next.js 字体加载方式，减少阻塞请求。

首期不使用小节虚拟化，因为不同小节高度、歌词输入焦点和导出完整谱面会显著增加复杂度。完成上述优化后再通过真机数据判断是否需要虚拟化。

## 9. 可访问性

- 可点击的音符和谱面元素使用 `button`，或补齐 `role`、键盘操作和可访问名称。
- 所有仅图标按钮提供中文 `aria-label`。
- 活动模式使用 `aria-pressed`，抽屉使用正确的 dialog 语义与焦点管理。
- 聚焦态保持可见，不因移动样式移除键盘操作能力。
- 颜色之外同时使用图标、文字或边框表达选择、禁用和错误状态。
- 遵循 `prefers-reduced-motion`，关闭非必要的位移和缩放动画。

## 10. 组件边界

建议新增或拆分以下组件：

- `MobileEditorShell`：移动端三段式布局及抽屉状态协调。
- `MobileEditorHeader`：顶部栏和全局操作。
- `MobileNoteBar`：常驻音符输入条。
- `MobileSelectionBar`：选中元素后的上下文操作。
- `MobileToolSheet`：完整编辑工具抽屉。
- `ScoreSettingsSheet`：乐谱设置表单。
- `EditorActionGroups`：抽取桌面面板和移动抽屉共享的动作定义、禁用规则与标签。

`ScoreEditor` 负责组装桌面和移动布局，两种布局共享 `ScoreCanvas`、`useScoreStore`、保存、导出和快捷键逻辑。移动组件不直接复制 store action 的业务实现。

## 11. 错误处理与状态恢复

- 保存和导出失败继续通过现有 Toast 展示，不阻塞谱面编辑。
- 导出资源动态加载失败时恢复导出按钮状态，允许用户重试。
- 抽屉打开状态不持久化；页面刷新后回到默认谱面视图。
- 显示比例、指法图和歌词显示设置继续沿用当前持久化策略。
- 离开页面时保持现有自动保存和返回前保存逻辑，不新增浏览器原生确认框。

## 12. 验证标准

### 12.1 静态与构建验证

- 运行现有编辑器单元测试。
- 运行 lint 和 production build。
- 确认桌面编辑器功能与布局没有回归。

### 12.2 手工验证尺寸

- 竖屏：360×800、390×844、430×932。
- 横屏：844×390。
- 平板：768×1024。
- 桌面：1280×800、1440×900。

### 12.3 核心流程

- 新建六孔和十二孔乐谱。
- 连续输入高、中、低音和休止符。
- 替换、插入、撤销、恢复和删除。
- 修改时值、附点、延长线、连音线和合并时值线。
- 输入中文歌词、切换第二行、批量粘贴。
- 编辑标题与乐谱信息。
- 自动保存、手动保存、返回前保存和导出图片。
- 软键盘弹出、收起以及横竖屏切换后界面仍可操作。

## 13. 实施顺序

1. 建立动态视口、安全区和响应式工作区骨架。
2. 实现移动顶部栏、常驻音符输入条和乐谱设置抽屉。
3. 实现选中元素上下文操作和完整工具抽屉。
4. 调整谱面触控区域、歌词输入和移动帮助交互。
5. 优化 store 订阅、导出依赖和字体加载。
6. 完成静态验证，并由项目维护者启动本地服务器进行真机与浏览器验证。
