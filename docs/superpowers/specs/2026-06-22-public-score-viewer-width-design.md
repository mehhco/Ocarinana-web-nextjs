# 乐谱广场详情页谱面宽度调整设计

## 目标

公开乐谱详情页中的谱面应充分利用现有查看器卡片宽度，避免固定 760px 容器在桌面端产生大面积留白。

## 方案

- 仅调整 `PublicScoreViewer` 内承载 `ScoreCanvas` 的容器宽度。
- 将固定宽度改为占满可用宽度，不扩大详情页现有的 `max-w-6xl` 页面边界。
- 保留查看器高度、滚动、缩放、歌词与指法图开关及图片导出行为。
- 不修改编辑器中的 `ScoreCanvas` 使用方式。

## 响应式行为

- 桌面端：谱面宽度铺满查看器卡片。
- 窄屏：沿用现有查看器溢出与滚动行为，不新增断点或固定最小宽度。

## 验证

- 运行 ESLint，确认 JSX 与 Tailwind 类名修改无静态检查问题。
- 不启动本地服务器，由项目维护者自行进行页面视觉验证。
