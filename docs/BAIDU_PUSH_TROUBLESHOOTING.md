# 百度推送API故障排除

## 当前测试结果

✅ **API路由已可访问** - 不再返回405错误
❌ **百度API返回400错误** - 需要检查配置

## 可能的原因和解决方案

### 原因1: 环境变量未配置

**检查方法：**
1. 登录 Vercel Dashboard
2. 进入项目 Settings → Environment Variables
3. 检查是否存在：
   - `BAIDU_PUSH_SITE`
   - `BAIDU_PUSH_TOKEN`

**如果未配置：**
- 按照之前的指南获取Token并配置
- 重新部署应用

### 原因2: Token错误

**检查方法：**
1. 在百度站长平台，进入 "数据引入" → "链接提交" → "API提交"
2. 确认Token是否正确复制（没有多余空格）
3. 确认Token是否过期

**解决方案：**
- 重新获取Token
- 更新Vercel环境变量
- 重新部署

### 原因3: Site URL格式错误

**检查方法：**
- 确认 `BAIDU_PUSH_SITE` 的值是：`https://www.ocarinana.com`
- 不要包含尾部斜杠
- 必须使用HTTPS

### 原因4: 百度API配额问题

**可能情况：**
- 今日推送配额已用完
- 新站点配额为0（需要填写备案号）

**解决方案：**
- 在百度站长平台填写备案号
- 或等待第二天配额重置

## 下一步操作

1. **检查环境变量配置**
   - 确认 `BAIDU_PUSH_SITE` 和 `BAIDU_PUSH_TOKEN` 已正确配置
   - 确认已重新部署

2. **如果环境变量未配置**
   - 这是正常的，因为这是可选功能
   - 如果需要使用，请按照 `docs/BAIDU_API_SETUP.md` 配置

3. **如果已配置但仍返回400**
   - 检查Token是否正确
   - 检查Site URL格式
   - 查看Vercel部署日志获取更详细的错误信息
